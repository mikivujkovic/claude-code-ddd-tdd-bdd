# Definition of Done (per phase)

## DDD phase (discovery done)
- [ ] Glossary exists and terms are consistent
- [ ] Bounded context described
- [ ] Model written (simple)
- [ ] Invariants listed for the next capability

## TDD phase (implementation done)
- [ ] Core domain rules implemented with unit tests
- [ ] Red/Green/Refactor practiced for each capability
- [ ] /verify passes (✅ VERIFY PASSED)
- [ ] No UI-driven tests for domain rules

## BDD phase (behavior done)
- [ ] Feature scenarios written for each capability
- [ ] Scenarios avoid UI mechanics; outcomes observable
- [ ] Scenario→service mapping written
- [ ] /verify passes (✅ VERIFY PASSED)
